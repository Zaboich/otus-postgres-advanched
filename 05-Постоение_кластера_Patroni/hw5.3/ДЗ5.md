# Отчёт о домашней работе 5 "Высокая доступность: развертывание Patroni"
При работе над ДЗ  был подготовлен bash скрипт [prep.sh](./prep.sh). Сприпт проводит все подготовительные работы:
1. Создаёт инфраструктуру для теста в Yandex Cloud: сеть, подсеть, 8 vm/
2. Устанавливает на vm машиные требуемые пакеты
    на 3 vm устанавливается ETCD и настраивает кластер etcd
    на 3 vm устанавливается Postgresql 18 и Patroni. Настраивается и запускается кластер Patroni
    на 1 vm устанавливается и настраивается балансировщик Haproxy для балансировки запросов от клиентов
    на 1 vm устанавливается клиент Postgres psql. Эта vm имитирует обращение клиентов к БД
3. После настройки и запуска кластера Patroni с тестовой машины в БД загружается дамп БД достаточно большого объёма 
4. Проверяется доступность данных из созданной БД demo. 
5. Останавливается сервис patroni на vm лидере кластера 
6. Проверяется состояние кластера. 
7. Повторно проверяется доступность данных в БД demo. Данные должны быть доступны благодаря автоматическому failover перестроению кластера patroni и обновлению данных о новом лидере кластера в Haproxy  
8. Запускает сервис patroni на бывшем лидере patroni. 
9. Проверяется состояние кластера. Бывший лидер должен подключится к кластеру и получить роль реплики.
10. Проводится ручное переключение лидера. Роль лидера передаётся vm изначально (при старте кластера) ставшей лидером. 
    Для этого определяется текущий лидер и выполняется команда patronictl switchover
11. Проверяется доступность данных после ручного переключения лидера


Скрипт запускался командой
```bash
./prep.sh | tee out_$(date +%F_%T).txt
```
многократно с проверкой результатов. Вывод сохранялся в файлы, например [out_2026-03-24_18-52-28.txt](./out_2026-03-24_18-52-28.txt) [](./out_2026-03-25_10-45-31.txt).
